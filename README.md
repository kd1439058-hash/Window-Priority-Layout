# Window Priority Layout

## 概要

Window Priority Layout は、Windows 上で開いているウィンドウを、あらかじめ設定した優先度とレイアウトに基づいて自動配置するデスクトップアプリケーションです。

ホットキーを押すだけで、現在開いているウィンドウを判定し、自動で指定した位置へ配置します。

## 制作背景

Windows 標準のスナップ機能や Microsoft PowerToys の FancyZones は便利ですが、どちらもユーザーが手動でウィンドウを配置する必要があります。

そこで、「現在開いているウィンドウを自動で判定し、優先度に応じて配置できれば、作業効率を向上できるのではないか」と考え、本アプリを制作しました。

## 特徴

- 現在開いているウィンドウを自動取得
- プロセス名・ウィンドウタイトルによる優先度判定
- 優先度順に自動ソート
- あらかじめ設定したレイアウトへ自動配置
- グローバルホットキーによる実行
- JSONによる設定保存
- タスクトレイ常駐

## 動作イメージ

1. 開いているウィンドウを取得
2. 優先度ルールを適用
3. 優先度順に並び替え
4. レイアウトへ割り当て
5. MoveWindow APIで配置

## 主な機能

### 優先度設定

- プロセス名またはタイトルで判定
- 優先度の編集
- JSON保存

### レイアウト設定

- 見本ウィンドウをドラッグして配置
- 座標・サイズを保存

### 自動配置

ホットキーを押すと

- ウィンドウ取得
- 優先度判定
- ソート
- レイアウト割り当て
- 自動配置

を実行します。

## 工夫した点

### タイトル判定への対応

同じアプリを複数起動した場合、プロセス名だけでは区別できません。

そのため、判定対象を

- プロセス名
- ウィンドウタイトル

から選択できるようにしました。

### 最大化・最小化への対応

最大化・最小化されたウィンドウは MoveWindow だけでは移動できません。

そこで ShowWindow を使用して通常状態へ戻した後に移動するよう実装しました。
