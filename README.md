# Window Priority Layout

## 概要

Window Priority Layout は、**ホットキー（またはボタン）一つで、起動中のアプリケーションウィンドウを優先度順に自動配置する Windows デスクトップアプリケーション**です。

既存のウィンドウ管理ツールでは、ユーザーがドラッグ＆ドロップなどで配置する必要があります。本アプリでは、起動中のウィンドウを自動取得し、あらかじめ設定した優先度に従ってレイアウトへ割り当てることで、ワンアクションで作業環境を再現できます。

## 制作背景

普段の作業では、Visual Studio、ブラウザ、エクスプローラーなど複数のアプリケーションを同時に使用しています。

Windows標準のスナップレイアウトや既存ツールは便利ですが、毎回手動で配置する必要があり、同じ作業を繰り返すことに不便さを感じました。

そこで、

* 起動中のウィンドウを取得
* 優先度を自動判定
* ボタン一つで指定レイアウトへ配置

を実現するツールを制作しています。

## 主な機能（予定）

* 起動中ウィンドウの取得
* ウィンドウタイトル・プロセス名の取得
* 優先度ルールによる自動判定
* 優先度順へのソート
* 指定レイアウトへの自動配置
* ホットキーによる実行
* 優先度設定画面（実装予定）

## システム構成

```
WindowCollector
        │
        ▼
WinInfo（Handle・Title・ProcessName）
        │
        ▼
WindowSorter
（優先度付与・並び替え）
        │
        ▼
LayoutManager
（配置先決定）
        │
        ▼
Win32 API
MoveWindow()
```

## 工夫した点

* ウィンドウ情報を `WinInfo` クラスへ集約し、取得処理と判定処理を分離
* 優先度ルールを `SortRule` クラスとして管理し、拡張しやすい設計
* ウィンドウ取得・優先度判定・配置処理をクラスごとに分割し、保守性を高めた
